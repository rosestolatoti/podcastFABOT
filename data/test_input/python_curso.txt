# Curso Completo de Python Fundamentos

## Introdução à Programação com Python

Python é uma linguagem de programação de alto nível, interpretada e multiparadigma, criada por Guido van Rossum em 1991. A linguagem é conhecida mundialmente por sua filosofia que enfatiza a legibilidade do código, utilizando indentação significativa para delimitar blocos de código. Python suporta múltiplos paradigmas de programação, incluindo programação orientada a objetos, programação imperativa e programação funcional.

A história do Python começa na Holanda, onde Guido van Rossum trabalhava no Centro de Matemática e Informática (CWI). Ele queria criar uma linguagem que fosse simples de aprender, fácil de usar e que permitisse prototipagem rápida. O nome Python não vem da serpente, mas sim do grupo de comédia britânico Monty Python, demonstrando o senso de humor dos criadores da linguagem.

Python se tornou uma das linguagens mais populares do mundo debido a várias características distintas. A sintaxe limpa e expressiva permite que программисты escribam menos código para fazer mais. A biblioteca padrão vasta e abrangente oferece módulos para praticamente qualquer tarefa imaginável. Além disso, a comunidade ativa e acolhedora cria bibliotecas especializadas para ciência de dados, aprendizado de máquina, desenvolvimento web e automação.

## Variáveis e Tipos de Dados

Variáveis em Python são containers para armazenar valores de dados. Diferentemente de muitas outras linguagens de programação, Python é dinamicamente tipado, o que significa que não é necessário declarar o tipo de uma variável antes de usá-la. O interpretador Python infere o tipo com base no valor atribuído.

O tipo inteiro (int) representa números inteiros sem parte decimal. Em Python 3, não há limite prático para o tamanho de inteiros, diferente de linguagens como C ou Java. Inteiros podem ser positivos ou negativos e suportam operações matemáticas básicas como adição, subtração, multiplicação e divisão.

O tipo float representa números de ponto flutuante, ou seja, números com parte decimal. Floats são essenciais para cálculos científicos, financeiros e qualquer aplicação que requer precisão decimal. Python usa representação de ponto flutuante binário, o que pode causar pequenas imprecisões em cálculos muito precisos.

O tipo string (str) representa sequências de caracteres. Strings em Python são imutáveis, o que significa que uma vez criadas, não podem ser modificadas. Esta característica traz benefícios em termos de performance e segurança. Strings podem ser criadas usando aspas simples ou duplas.

O tipo booleano (bool) representa valores de verdade: True ou False. Booleanos são fundamentais para controle de fluxo e expressões lógicas. Em Python, True e False são casos especiais de inteiros, onde True é equivalente a 1 e False a 0.

Listas são coleções ordenadas e mutáveis de elementos. Uma lista pode conter elementos de diferentes tipos, incluindo outras listas. Listas são criadas usando colchetes e elementos são separados por vírgulas. O acesso aos elementos é feito através de índices, começando em zero.

Tuplas são similares a listas, mas são imutáveis. Once created, a tuple cannot be modified. Tuplas são usadas para representar coleções fixas de elementos, como coordenadas geográficas ou registros de banco de dados. A imutabilidade traz benefícios em termos de performance e permite que tuplas sejam usadas como chaves de dicionários.

Dicionários são coleções de pares chave-valor. Cada chave única está associada a um valor correspondente. Dicionários são extremamente versáteis e permitem acesso rápido aos valores através de suas chaves. Esta estrutura de dados é fundamental em Python e é usada extensivamente em todo tipo de aplicação.

## Operadores e Expressões

Operadores aritméticos em Python incluem adição (+), subtração (-), multiplicação (*), divisão (/), divisão inteira (//), módulo (%) e exponenciação (**). A divisão em Python 3 sempre retorna um float, mesmo quando os operandos são inteiros. A divisão inteira (//) retorna apenas a parte inteira do resultado.

Operadores de comparação comparam valores e retornam booleanos. Igualdade é verificada com (==), diferença com (!=), maior que (>), menor que (<), maior ou igual (>=) e menor ou igual (<=). Estes operadores podem ser encadeados em Python, permitindo expressões como 1 < x < 10.

Operadores lógicos (and, or, not) combinam ou invertem valores booleanos. O operador and retorna True apenas se ambos os operandos forem verdadeiros. O operador or retorna True se pelo menos um operando for verdadeiro. O operador not inverte o valor booleano.

Operadores de atribuição estendem a atribuição básica (=). Atribuição com adição (+=), subtração (-=), multiplicação (*=) e assim por diante combinam a operação com a atribuição em uma única expressão. Estes operadores tornam o código mais conciso e podem ser mais eficientes.

## Controle de Fluxo

Estruturas condicionais permitem que o programa tome decisões baseadas em condições. O if é a estrutura condicional mais básica, executando um bloco de código apenas se a condição for verdadeira. O elif permite testar múltiplas condições em sequência. O else captura todos os casos não capturados pelas condições anteriores.

A indentação em Python é crucial e faz parte da sintaxe da linguagem. Blocos de código são definidos por seu nível de indentação, não por chaves ou palavras-chave. Esta característica enforce a legibilidade do código e previne erros comuns encontrados em outras linguagens.

Loops permitem repetir blocos de código. O loop for itera sobre uma sequência (lista, tuple, string, ou range). O loop while continua iterando enquanto uma condição for verdadeira. Ambos os loops podem ser controlados com break (interromper) e continue (pular para próxima iteração).

A função range() gera uma sequência de números. range(n) gera números de 0 a n-1. range(início, fim) gera números de início a fim-1. range(início, fim, passo) permite especificar o incremento. Range é frequentemente usado com loops for para controle de iterações.

List comprehensions oferecem uma forma concisa e pythonica de criar listas. A sintaxe [expressão for item in iterável if condição] permite criar listas baseadas em outros iteráveis de forma elegante e eficiente. List comprehensions são preferidas sobre loops tradicionais quando apropriadas.

## Funções

Funções são blocos de código reutilizáveis que realizam uma tarefa específica. Em Python, funções são definidas usando a palavra-chave def seguida do nome da função e parênteses contendo os parâmetros. O código da função é indentado e executado quando a função é chamada.

Parâmetros de função podem ter valores padrão, tornando-os opcionais na chamada. Argumentos podem ser passados por posição ou por nome. Python também suporta número variável de argumentos (*args para posicionais, **kwargs para nomeados).

O comando return envia um valor de volta ao chamador. Se nenhuma expressão return for fornecida ou se a função atingir o fim sem return, a função retorna None. Funções podem ter múltiplos pontos de return em diferentes branches do código.

Escopo de variáveis determina onde uma variável pode ser acessada. Variáveis definidas dentro de uma função são locais e não afetam variáveis com o mesmo nome fora da função. A palavra-chave global permite modificar variáveis no escopo global de dentro de uma função.

Funções lambda são funções anônimas definidas em uma única linha. A sintaxe lambda parâmetros: expressão cria uma função sem nome que pode ser atribuída a uma variável ou passada como argumento. Lambdas são úteis para operações simples e callbacks.

Decoradores são funções que modificam o comportamento de outras funções. A sintaxe @decorator antes de uma definição de função aplica o decorador. Decoradores são comumente usados para logging, verificação de tipos, caching e controle de acesso.

## Programação Orientada a Objetos

Classes são modelos para criar objetos. Uma classe define atributos (dados) e métodos (funções) que operam nesses dados. Objetos são instâncias de classes, cada um com seu próprio estado.

O método __init__ é o inicializador de uma classe, chamado quando um objeto é criado. O parâmetro self representa a instância do objeto e deve ser o primeiro parâmetro de todos os métodos.

Herança permite criar novas classes baseadas em classes existentes. A classe filha herda todos os atributos e métodos da classe pai e pode adicionar novos ou sobrescrever os existentes. Python suporta herança múltipla.

Encapsulamento é a prática de ocultar detalhes internos de uma classe. Python usa convenções de nomenclatura: atributos começando com underscore (_) são considerados protegidos, e com double underscore (__) são considerados privados.

Polimorfismo permite que objetos de diferentes classes sejam tratados de forma uniforme através de uma interface comum. Em Python, o polimorfismo é dinâmico e baseado em duck typing, onde o que importa é se um objeto tem os métodos necessários, não sua classe específica.

## Manipulação de Erros

Exceções são eventos que ocorrem durante a execução e interrompem o fluxo normal do programa. Python usa um modelo de tratamento de exceções onde erros são representados como objetos de exceção que podem ser capturados e tratados.

O bloco try/except captura e trata exceções. Código que pode gerar exceções é colocado no bloco try. Código que trata exceções específicas é colocado no bloco except. Múltiplos except podem capturar diferentes tipos de exceção.

O bloco finally é sempre executado, independentemente de exceções ocorrerem ou não. É usado para código de limpeza, como fechar arquivos ou conexões de banco de dados.

Levantando exceções com raise permite indicar erros condições no código. raise Exception("mensagem") cria e levanta uma exceção genérica. Exceções customizadas podem ser criadas herdando de Exception.

## Módulos e Pacotes

Módulos são arquivos Python contendo definições e instruções. O comando import carrega um módulo e permite acesso suas funções, classes e variáveis. A sintaxe from módulo import nome importa itens específicos.

A biblioteca padrão Python é extremamente vasta e inclui módulos para operação de arquivos, rede, data e hora, matemática, criptografia e muito mais. Alguns módulos importantes incluem os para operação de sistema, datetime para manipulação de datas, json para serialização de dados e re para expressões regulares.

Pacotes são diretórios contendo módulos e um arquivo __init__.py. A estrutura de pacotes permite organizar código em namespaces hierárquicos. PIP, o gerenciador de pacotes Python, permite instalar pacotes de terceiros.

## Estruturas de Dados Avançadas

### Sets (Conjuntos)

Sets são coleções não ordenadas de elementos únicos. Em Python, sets são criados usando chaves {} ou a função set(). A principal característica dos sets é que elementos duplicados são automaticamente eliminados. Sets são extremamente úteis para eliminar duplicatas e realizar operações de teoria de conjuntos como união, interseção e diferença.

A operação de união (| ou union()) combina elementos de dois sets. A operação de interseção (& ou intersection()) retorna apenas elementos presentes em ambos os sets. A operação de diferença (- ou difference()) retorna elementos presentes no primeiro set mas não no segundo.

Sets não suportam indexação ou fatiamento porque são estruturas não ordenadas. No entanto, membership testing (verificar se um elemento está no set) é muito rápido, com complexidade O(1) em média.

### Frozensets

Frozensets são como sets, mas imutáveis. Uma vez criado, um frozenset não pode ser modificado. Esta característica permite que frozensets sejam usados como elementos de outros sets ou como chaves de dicionários.

### Collections

O módulo collections fornece estruturas de dados especializadas além das built-in. Counter é um dict para contar elementos hashable. OrderedDict lembra a ordem em que elementos foram adicionados. DefaultDict fornece valores default para chaves inexistentes. NamedTuple cria tuplas com campos nomeados. Deque é uma fila dupla eficiente para inserções e remoções em ambas as extremidades.

### Data Classes

Data classes, disponíveis a partir do Python 3.7, são uma forma concisa de criar classes que servem principalmente para armazenar dados. O decorador @dataclass gera automaticamente métodos como __init__, __repr__ e __eq__.

## Programação Funcional

### Funções de Alta Ordem

Funções de alta ordem são funções que recebem funções como argumentos ou retornam funções.map() aplica uma função a cada elemento de um iterável. filter() seleciona elementos baseado em uma função predicado. reduce() reduz um iterável a um único valor acumulativo.

### Comprehensions

Além de list comprehensions, Python oferece dict comprehensions e set comprehensions. A sintaxe é similar: {chave: valor for item in iterável} para dicionários e {expressão for item in iterável} para sets.

Generator expressions são como list comprehensions mas com parênteses. Eles produzem elementos sob demanda (lazy evaluation), sendo mais eficientes em memória para sequências grandes.

### Iteradores

Iteradores são objetos que implementam os métodos __iter__ e __next__. Iteráveis são objetos que podem ser passados para iter() e retornam um iterador. Listas, tuples, strings, sets e dicionários são todos iteráveis.

A função iter() retorna um iterador de um iterável. A função next() retorna o próximo elemento do iterador. Quando não há mais elementos, next() levanta StopIteration.

### Geradores

Geradores são funções que usam yield para retornar elementos um de cada vez. Diferentemente de retornar uma lista, geradores produzem elementos sob demanda, sendo muito eficientes em memória. Generator functions são especialmente úteis para processar grandes volumes de dados ou sequências infinitas.

## Trabalhando com Arquivos

### Leitura e Escrita

A função open() abre um arquivo e retorna um objeto file. O primeiro argumento é o nome do arquivo. O segundo argumento é o modo: 'r' para leitura, 'w' para escrita (sobrescreve), 'a' para append, 'b' para binário.

É importante sempre fechar arquivos após o uso. A forma recomendada é usar o context manager (with), que garante que o arquivo seja fechado mesmo se ocorrerem exceções.

### Modos de Arquivo

'r' abre para leitura (default). 'w' abre para escrita, criando um novo arquivo ou sobrescrevendo existente. 'x' cria um novo arquivo e abre para escrita, falhando se existir. 'a' abre para append, criando se não existir. Modos podem ser combinados com 'b' para binário ou '+' para leitura e escrita.

### Lendo Linhas

read() lê todo o arquivo ou número especificado de bytes. readline() lê uma linha. readlines() lê todas as linhas e retorna uma lista. Alternativamente, iterar sobre o objeto file linha por linha é mais eficiente em memória.

### Arquivos JSON

O módulo json permite serializar dados Python para JSON e deserializar JSON para dados Python. dump() escreve para arquivo. load() lê de arquivo. dumps() retorna string. loads() parsing de string.

## Expressões Regulares

Expressões regulares são padrões usados para encontrar e manipular texto. O módulo re fornece funções para trabalhar com regex.

O padrão mais simples é uma sequência literal de caracteres que corresponde a si mesma. Metacaracteres como . (qualquer caractere), \d (dígito), \w (caractere de palavra), \s (espaço em branco) permitem padrões mais flexíveis.

Quantificadores especificam quantas vezes um padrão pode ocorrer. * significa zero ou mais. + significa um ou mais. ? significa zero ou um. {n} significa exatamente n. {n,m} significa entre n e m.

Agrupamentos com parênteses capturam partes do padrão. Referências anteriores permitem fazer referência a grupos capturados. A alternância com | permite escolher entre padrões.

## Context Managers e o Protocolo with

Context managers garantem que recursos sejam adquiridos e liberados corretamente. O protocolo with gerencia entrada e saída de contexto através dos métodos __enter__ e __exit__.

A função contextlib fornece ferramentas para criar context managers. O decorator @contextmanager permite criar context managers usando geradores. Isso simplifica significativamente a criação de context managers customizados.

## Testes

O módulo unittest fornece framework para testes unitários. TestCase é a classe base para casos de teste. Métodos que começam com test_ são executados como testes.

Assertions verificam condições. assertEqual() verifica igualdade. assertTrue() verifica verdade. assertRaises() verifica que uma exceção é levantada.

Setup e teardown permitem executar código antes e depois de cada teste. setUp() é executado antes de cada método de teste. tearDown() é executado após cada método de teste.

## Decoradores Avançados

Decoradores são funções que recebem uma função e retornam uma nova função. Decoradores com argumentos usam uma factory de decoradores.

functools.wraps preserva metadados da função original quando decorada. O decorator @wraps do módulo functools é essencial para criar decoradores bem comportados.

## Metaclasses

Metaclasses são classes cujas instâncias são classes. O padrão padrão de metaclasses em Python é o type. Custom metaclasses permitem controlar a criação de classes.

O método __new__ de uma metaclasse é chamado antes de __init__ e é responsável por criar a nova classe. __init__ inicializa a classe após sua criação.

## Descriptors

Descriptors são objetos que implementam os métodos __get__, __set__ ou __delete__. Eles permitem controlar o acesso a atributos de forma granular.

Property é um decorator que cria um descriptor simples. Funções getter, setter e deleter definem o comportamento de leitura, escrita e exclusão do atributo.

## Context Variables

Contexto de variáveis são variáveis que têm valores diferentes em diferentes contextos de execução. O módulo contextvars fornece tools para working with context variables em asyncio e threading.

## Async e Await

Programação assíncrona permite que operações lentas não bloqueiem a execução. async def define uma corrotina. await suspende a execução até que a corrotina seja completada.

O módulo asyncio fornece infrastructure para escrever código assíncrono. Event loop gerencia a execução de corrotinas. Tasks representam corrotinas agendadas para execução.

## Type Hints

Type hints permitem annotar tipos de variáveis, parâmetros e valores de retorno. Type hints melhoram documentação e permitem ferramentas de análise estática.

O módulo typing fornece types genéricos como List, Dict, Set, Tuple. Callable representa tipos de funções. Optional representa valores que podem ser None. Union representa tipos múltiplos.

## Compressed Files

Módulos como gzip, bz2 e zipfile permitem ler e escrever arquivos comprimidos. gzip.open() abre arquivos gzip. zipfile.ZipFile() working com arquivos ZIP.

## Logging

O módulo logging fornece framework flexível para logging messages. Logger é o objeto principal que cria log messages. Handlers enviam logs para diferentes destinations como arquivos, console ou rede.

Formatters especificam o formato das mensagens de log. Níveis de log (DEBUG, INFO, WARNING, ERROR, CRITICAL) controlam quais mensagens são registradas.

## Environment Variables

O módulo os fornece access a environment variables através de os.environ. Environment variables são úteis para configuração de aplicações e separation of concerns.

## Virtual Environments

Virtual environments permitem criar ambientes Python isolados com suas próprias dependências. venv é o módulo built-in para criar virtual environments. pip instala packages no environment ativo.

## Pacotes e Distribuição

setup.py e pyproject.toml definem metadata sobre packages. O módulo distutils e setuptools fornece tools para distribuir packages. wheel é o formato de distribuição binária padrão.

## Performance Optimization

Perfis de código identificam gargalos de performance. O módulo cProfile fornece profiling determinístico. timeit mede tempo de execução de pequenas partes de código.

Algoritmos mais eficientes e estruturas de dados apropriadas são frequentemente mais importantes que micro-otimizações. Uso de generators ao invés de listas pode reduzir significativamente o uso de memória.

## Security Considerations

Nunca use eval() ou exec() com input não confiável. Sanitize todos os inputs antes de usar em queries ou comandos. Mantenha dependências atualizadas para evitar vulnerabilidades conhecidas.

O módulo secrets fornece functions para gerar cryptograficamente strong random numbers. Use hashing para armazenar senhas, nunca armazene passwords em plaintext.

## Best Practices

Código limpo segue convenções de estilo (PEP 8). Docstrings documentam módulos, classes e funções. Comments explicam o "porquê", não o "quê".

Princípios SOLID aplicam-se a Python orientado a objetos. KISS (Keep It Simple, Stupid) sugere simplicidade. DRY (Don't Repeat Yourself) evita duplicação. YAGNI (You Aren't Gonna Need It) evita over-engineering.
