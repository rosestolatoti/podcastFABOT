                                                        |
                                                        v
                              API JOBS/START <- CRIA JOB NO BANCO
                                                        |
                                                        v
                              WORKER ASSÍNCRONO <- INICIA PROCESSAMENTO
                                                        |
                        +---------------+---------------+---------------+
                        |               |               |               |
                        v               v               v               v
                   LEITURA       LLM (GROQ)       TTS (KOKORO)   POST-PROD
                   (PENDING)   (LLM_PROCESSING) (TTS_PROCESSING) (DONE)
                        |               |               |               |
                        v               v               v               v
                   PENDING -> READING -> SCRIPT_DONE -> DONE
                                                        |
                                                        v
                              FRONTEND (POLLING) <- RETORNA AUDIO_PATH
                                                        |
                                                        v
                              PLAYER EXIBE ÁUDIO

5. PROCESSAMENTO NO BACKEND (WORKER)
================================================================================

ARQUIVO: backend/workers/podcast_worker.py

FUNÇÃO: process_podcast_job(job_data, job_id)

ETAPAS DO PROCESSAMENTO:

1. READING (progress: 5%)
   - Lê o texto do banco (script_json)
   - Ou extrai texto de PDF/DOCX

2. LLM_PROCESSING (progress: 20-40%)
   - Chama Groq API com o texto
   - Modelo: llama-3.3-70b-versatile
   - Gera roteiro em formato JSON:
     {
       "title": "Título",
       "segments": [
         {
           "speaker": "pm_alex",
           "text": "Texto do segmento",
           "emotion": "neutral|enthusiastic|serious",
           "pause_after_ms": 600
         }
       ]
     }
   - Salva no banco (script_json)

3. TTS_PROCESSING (progress: 45-85%)
   - Para cada segmento do roteiro
   - Chama Kokoro TTS API
   - Endpoint: https://api.kokoro.ai/v1/audio/speech
   - Voz: pm_alex (PT-BR masculino)
   - Gera arquivos MP3 individuais
   -拼接todos em um arquivo único

4. POST_PRODUCTION (progress: 85-100%)
   - Normalização de áudio (LUFS)
   - Aplicar filtros
   - Mesclar intro/outro (música)
   - Salvar arquivo final em data/output/

         - setActiveTab('progresso')

           - GET /jobs/{job_id}

16. CORREÇÕES FEITAS EM 2026-03-15
================================================================================

PROBLEMA ORIGINAL:
O endpoint /jobs/{id}/start era SÍNCRONO e BLOQUEAVA o servidor HTTP.
Quando o frontend chamava /start, o backend processava tudo (LLM + TTS + 
pós-produção) dentro da própria requisição HTTP antes de responder. O polling 
do frontend ficava fazendo GET /jobs/{id} a cada 2 segundos, mas o backend 
estava travado processando e não respondia ao polling até terminar. Por isso 
o progresso ficava em 0% — ele só ia para 100% quando o /start finalmente 
retornava.

SOLUÇÃO IMPLEMENTADA:

--------------------------------------------------------------------
CORREÇÃO 1: BackgroundTasks no /start
--------------------------------------------------------------------
ARQUIVO: backend/routers/jobs.py

ALTERAÇÕES:
1. Adicionado import: BackgroundTasks do FastAPI
2. Adicionado import: SessionLocal do database
3. Criada função wrapper: run_podcast_job_background(job_id)
   - Cria sua própria sessão DB
   - Executa o processamento em loop de eventos separado
   - Trata erros e atualiza status para FAILED se necessário

4. Modificado endpoint /start:
   ANTES (síncrono):
   ```python
   @router.post("/{job_id}/start")
   async def start_job(job_id: str, db: Session = Depends(get_db)):
       result = await process_podcast_job({}, job_id)  # BLOQUEIA ATÉ TERMINAR
       return {"status": "processing", "job_id": job_id, "result": result}
   ```

   DEPOIS (assíncrono):
   ```python
   @router.post("/{job_id}/start")
   async def start_job(job_id: str, background_tasks: BackgroundTasks, db: Session = Depends(get_db)):
       job.status = "READING"
       job.progress = 5
       job.current_step = "Processando..."
       db.commit()
       
       background_tasks.add_task(run_podcast_job_background, job_id)
       
       return {"status": "queued", "job_id": job_id}  # RETORNA IMEDIATAMENTE
   ```

RESULTADO:
- O endpoint agora retorna em MILISSEGUNDOS
- O processamento roda em background
- O polling do frontend funciona porque o servidor HTTP não está mais bloqueado

TESTE REALIZADO:

CORREÇÃO 2: Endpoint /cancel
--------------------------------------------------------------------
ARQUIVO: backend/routers/jobs.py

NOVO ENDPOINT:
```python
@router.post("/{job_id}/cancel")
async def cancel_job(job_id: str, db: Session = Depends(get_db)):
    job = db.query(Job).filter(Job.id == job_id).first()
    if not job:
        raise HTTPException(status_code=404, detail="Job não encontrado")
    
    if job.status in ["DONE", "FAILED"]:
        raise HTTPException(status_code=400, detail=f"Job já finalizado: {job.status}")
    
    job.status = "FAILED"
    job.error_message = "Cancelado pelo usuário"
    db.commit()
    
    return {"status": "cancelled", "job_id": job_id}
```

FUNÇÃO:
- Permite cancelar jobs em andamento
- Atualiza status para FAILED
- Define mensagem de erro "Cancelado pelo usuário"
- Retorna erro se job já estiver finalizado

   JOB_ID=$(echo $JOB | python3 -c "import sys,json; print(json.load(sys.stdin)['job_id'])")
   
   # Iniciar (retorna imediatamente)
   curl -s -X POST "http://localhost:8000/jobs/$JOB_ID/start"
   # Esperado: {"status":"queued","job_id":"..."}
   
   # Ver progresso (irá advancing)
   sleep 5 && curl -s "http://localhost:8000/jobs/$JOB_ID"
   # Esperado: LLM_PROCESSING, 20%
   
   sleep 15 && curl -s "http://localhost:8000/jobs/$JOB_ID"
   # Esperado: TTS_PROCESSING, 45%
   
   sleep 45 && curl -s "http://localhost:8000/jobs/$JOB_ID"
   # Esperado: DONE, 100%
